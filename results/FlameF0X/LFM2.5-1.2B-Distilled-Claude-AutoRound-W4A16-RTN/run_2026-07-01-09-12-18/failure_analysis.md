## Root Cause

During RTN blockwise quantization with importance matrix (W4A16 RTN), auto-round's _sampling_inputs function attempts to subsample calibration inputs by index, but the indices exceed the length of one or more tensors in input_others. This indicates the blockwise forward hooks captured inputs with inconsistent lengths across keys (e.g., input_ids has N samples while attention_mask/position_ids/labels has fewer), which is a known fragility in auto-round's RTN-with-imatrix path when used with newer/hybrid architectures like LFM2.5 that produce extra per-block inputs via hooks.
