## Root Cause

auto_round's _get_dataset_impl tokenizes NeelNanda/pile-10k without padding/truncation that would produce an attention_mask column, then blindly attempts set_format with both 'input_ids' and 'attention_mask'. The tokenizer call does not request return_attention_mask=True and the dataset has no pad token configured, so only 'input_ids' is added; the subsequent set_format call then fails on the missing column.
