## Root Cause

The 12B parameter model cannot fit in the 22.15 GiB GPU. The tokenizer loaded successfully (HTTP 200/302 responses), and the OOM occurs during model weight materialization when transformers tries to move tensors to GPU. A 12B model in BF16/FP16 needs ~24 GiB just for weights, exceeding available VRAM before quantization buffers are allocated.

## Suggested Fix

1) Move model loading off-GPU: pass device_map='cpu' or 'auto' with offload_folder='/tmp/offload' and offload_state_dict=True to AutoModelForCausalLM.from_pretrained, then let auto_round quantize layer-by-layer. 2) Use a GPU with ≥48 GiB (A6000/A100 80GB) for 12B class models. 3) Set PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True to reduce fragmentation (suggested by error). 4) Run on a smaller proxy model first, or use --device cpu flag if auto_round supports it for W4A16 RTN.
