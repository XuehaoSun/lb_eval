## Suggested Fix

Use multi-GPU sharding or CPU/NVMe offload during quantization. Concretely, in the quantize phase call: AutoModelForCausalLM.from_pretrained(..., device_map='auto', torch_dtype=torch.bfloat16) and AutoRound(..., device_map='auto') so the 27B model is split across available GPUs. If only one GPU is available, set device_map='cpu' (slow but works) or device_map='cuda' with low_cpu_mem_usage=True and PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True, and quantize with --nsamples 16 and --seqlen 512 to reduce activation memory. If a multi-GPU host exists, set CUDA_VISIBLE_DEVICES=0,1,2,3 and use device_map='auto'. For very tight VRAM, use the AutoRound checkpoint sharding / save while loading flow, or convert to GGUF:Q4_K_M via an external llama.cpp path.

## Workaround

If multi-GPU is unavailable, offload to CPU: pass device_map='cpu' to AutoModelForCausalLM.from_pretrained and AutoRound(... device_map='cpu'). Alternatively, quantize via llm-compressor/NVFP4 in a streaming/chunked fashion, or first dump the model to GGUF on CPU using a tool that does not require GPU-resident weights.
